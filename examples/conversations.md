# 示例对话 — Visual Cognition Slides 实际运作

这个文件展示 skill 在不同场景下的完整对话流程。
每个示例说明 AI 做了哪些判断，以及背后的理论依据。

---

## 示例 1：入口 A（粗想法）× 口播/社媒

**场景**：小红书博主，想做一期关于"为什么老师比任何时候都更重要"的口播视频

---

**用户：**
我想做一个关于"AI时代老师更重要"的口播，发小红书，大概5张slides。

**AI：**
先确认一件事——听完这期内容，你希望观众有什么变化？

是"觉得老师很重要，应该被尊重"，还是"作为老师，重新看见自己的不可替代性"？

**用户：**
第二个。我的受众主要是对AI有焦虑感的老师。

**AI：**
明白了。内容简报：

- 受众心理起点：焦虑（"AI会取代我吗"）
- 目标转变：焦虑 → 看见自己真正的不可替代性
- 核心信息：AI做不到的那件事，正好是教师最擅长的
- 推荐结构：**身份认同型**——"你也有这个焦虑 → 其实这是你的优势"
- 钩子句候选：「AI时代，老师不是要被替代，是要被需要得更深」

方向对吗？确认后选风格。

**用户：**
对，就是这个感觉。

**AI：**
视觉风格和画布格式选一下：

风格：① 手绘创意 ② 学术简约 ③ 商务专业 ④ 深色极简 ⑤ 暖色插画 ⑥ 科研极客 ⑦ 3D立体 ⑧ 纯动画概念
格式：横版1920×1080 / 竖版手机 / 小红书4:5 / 方图

**用户：**
手绘创意，小红书4:5。

**AI：**
5张slide结构：

| # | 认知动作 | Layout | 内容 |
|---|---------|--------|------|
| 1 | 建立代入 | 大字冲击 | 钩子句 |
| 2 | 激活焦虑（共情） | 对比型 | AI能做的 vs "那老师还有什么？" |
| 3 | 核心转折 | 左文右动画 | "AI没有情境，没有关系，没有判断" |
| 4 | 具体化优势 | 卡片矩阵 | 看见情绪 / 建立信任 / 定义什么值得学 |
| 5 | 行动收尾 | 大字冲击 | 金句 + CTA |

生成 HTML？

---

> **AI 在这里做的隐形判断：**
> - 选"身份认同型"结构 → 叙事传输理论：受众进入叙事，抵触降低
> - S2 用对比型而不是列表 → 对比激活认知张力，比陈述更有力
> - S3 用动画而不是文字 → 双通道：口播说"AI没有"，slide 动画演示"有关系的人的存在"
> - 竖版布局自动调整卡片为1列 → FORMATS.md 竖版规则

## 示例 2：入口 B（已有口播稿）× 微课

**场景**：大学教师，有写好的认知负荷讲稿，要转成slides用于线上课

---

**用户：**
我有这个脚本，帮我做成slides：

今天讲认知负荷理论。认知负荷就是大脑处理信息时的"压力"。就像手机内存满了会卡顿，大脑也一样。认知负荷分三种：内在负荷（内容本身的难度）、外在负荷（呈现方式造成的额外负担）、生成负荷（学习者主动建构理解的努力）。好的教学设计核心是降低外在负荷，保留生成负荷。举个例子：几何题如果图文分离，学生要不断转移视线，这就是外在负荷。如果注释直接放在图上，外在负荷消失，认知资源全部用在理解上。

**AI：**
快速诊断：这是**概念性知识**，结构清晰——定义 → 分类 → 原则 → 案例。

风格和画布先选一下？

**用户：**
学术简约，横版。

**AI：**
7张slides：

| # | 内容 | Layout | 动画 |
|---|------|--------|------|
| 1 | 封面：「你的PPT可能正在占用学生的大脑内存」 | 大字冲击 | 无 |
| 2 | 核心定义 + 手机卡顿类比 | 左文右图 | 手机内存填满动画 |
| 3 | 三种负荷 | 卡片矩阵（3张） | 逐卡出现 |
| 4 | 关键原则 | 大字冲击 | 「降低外在负荷，保留生成负荷」 |
| 5 | 反例（图文分离） | 对比左侧 | 视线跳动箭头动画 |
| 6 | 正例（注释在图上） | 对比右侧 | 注释就位动画 |
| 7 | 总结 + 你现在能做的 | 卡片矩阵 | — |

注意：三种负荷的学术名称做成 Space Mono 小标签，不放大——标签，不是主角。主角是日常语言解释。

生成？

---

> **AI 在这里做的隐形判断：**
> - 跳过了 Intake（入口B，已有稿子），直接诊断
> - 把"反例→正例"拆成两张，而不是一张对比 → 每张一个认知单元（认知负荷原则）
> - 学术名词降级为小标签 → 先验知识激活：用日常语言搭桥，专业术语是标注
> - S2 的类比用动画演示 → 手机内存填满比静态图更有冲击感

## 示例 3：入口 C（指定格式）× 报告

**场景**：EdTech 研究者，有用户访谈数据，要做汇报 slides

---

**用户：**
帮我做一个报告slides，主题是"教师在课堂中使用AI的障碍"，我有5个主要发现，受众是学校管理层，科研极客主题，横版。

**AI：**
好，报告结构遵循**结论先行**原则——管理层没时间看推导过程，最重要的发现放第一张。

请把5个发现发给我。

**用户：**
1. 教师对AI输出的准确性缺乏信任（78%受访者提及）
2. 现有AI工具与课程标准脱节
3. 技术支持不足，问题无处反馈
4. 数据隐私顾虑阻碍工具采纳
5. 成功案例不可见，缺乏同伴学习

**AI：**
12张slides结构：

```
S01 封面 + 最关键结论（78%信任问题）
S02 研究背景（一句话）+ 方法（一句话）
S03 发现总览（5条，动画逐条出现）
S04 发现1：信任缺口（数据 + 根本原因）
S05 发现2：课程脱节（具体案例引用）
S06 发现3：技术支持（现状 vs 需求对比）
S07 发现4：隐私顾虑（政策现状）
S08 发现5：同伴学习缺失
S09 综合分析：5个障碍的关系图
S10 建议（3条，有优先级）
S11 下一步行动（带时间节点）
S12 附录入口
```

S09的关系图用**动态连接图**——5个节点先各自出现，再用动画画出它们之间的因果连线，最后高亮最核心的那个节点。

生成？

---

> **AI 在这里做的隐形判断：**
> - 报告用"结论先行"，不是"铺垫→结论" → 逆向设计：管理层的目标是决策，不是学习
> - S09 用动态连接图而不是列表 → 关系性知识的最佳表达方式（PEDAGOGY.md §2.4）
> - 科研极客主题下字号更小、密度更高 → STYLES.md 主题6规则

## 示例 4：PPTX 转换 + 风格升级

**场景**：老师有一份已有PPT，想升级成HTML，同时改变风格

---

**用户：**
我有一份做了两年的数学课件PPT，想转成HTML，但希望风格更适合高中生，现在太无聊了。

**AI：**
好，两步走：

第一步先转换——把PPT发给我，或者运行：
```bash
python3 scripts/pptx_to_html.py 数学课件.pptx
```

这会生成一个基础HTML，保留所有文字和图片。

第二步风格升级——基础转换完成后，告诉我：
- 哪几张slide你觉得最重要？
- 有没有哪个概念你觉得"用文字解释了很久学生还是不懂"？

这些地方我帮你换成动画表达。

**用户：**
[上传了pptx文件]
第3张（函数定义）和第7张（图像变换）学生总是搞不清楚。

**AI：**
转换完成。看了这两张：

第3张"函数定义"是**概念性知识**，现在是纯文字。建议换成：
→ 类比动画（自动售货机：投入→处理→输出，对应 x→f→y）

第7张"图像变换"是**程序性知识**，现在是静态图。建议换成：
→ 几何变换动画（原函数用实线，变换后用虚线，动画演示平移/拉伸过程）

其他slides保持原内容，换成**暖色插画**主题（高中生更友好）。

要我直接生成升级版？

## 理论 × 操作 对照表

供想了解背后逻辑的人参考：

| AI 的操作 | 背后的理论 |
|-----------|-----------|
| 先问"听完要做/感受什么" | 逆向设计（McTighe）：终点决定路径 |
| 一张 slide 一个认知单元 | 认知负荷理论（Sweller） |
| 动画讲一件事，口播讲另一件事 | 双通道理论（Mayer） |
| 每个新概念配日常类比 | 先验知识激活（Schema Theory） |
| 故事/案例优先于抽象定义 | 叙事传输理论（Green & Brock） |
| 钩子句制造信息差 | 好奇缺口理论（Loewenstein） |
| 报告结论先行 | 逆向设计 × 受众目标分析 |
| 3D翻转展示"误解→真相" | 对比学习 + 先验错误激活 |
| 数据用面积/数量而非纯数字 | 具身认知 × 比例直觉 |
| 微课末尾加 Quiz | 检索练习效应（Roediger & Karpicke） |
